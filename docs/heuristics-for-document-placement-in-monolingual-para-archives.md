# Heuristics for Determining Document Placement in a Monolingual PARA Archive

## Introduction

This document explains how to infer the correct PARA parent directory for a document stored within a monolingual `en-ca` SAT archive.
The guidance relies on observable document characteristics, lifespan indicators, and known archival patterns.
The goal is predictable placement without relying on semantics or tooling assumptions.

This approach is informed by research on personal knowledge management and information architecture.

For example, the PARA method emphasizes the separation of actionable, ongoing, reference, and historical materials to support efficient retrieval <a name="forte-2022-citation"></a>[(Forte, 2022)](#forte-2022-reference).

## PARA Overview

PARA consists of four physical namespaces:

```
projects/
areas/
resources/
archives/
```

Each namespace has distinct qualities based on document activity, lifespan, and reuse patterns.

## Guessing Project Placement

Documents often belong in `projects/` when they exhibit signs of:

- goals, deliverables, or planned outcomes
- milestones or timelines
- iterative drafts or change logs
- active evolution during a bounded timeframe

These match the definition of “finite, outcome-oriented work structures” described in productivity and knowledge-system literature <a name="allen-2001-citation"></a>[(Allen, 2001)](#allen-2001-reference).

## Guessing Area Placement

Documents typically belong in `areas/` when they support:

- ongoing responsibilities
- continuous maintenance
- recurring updates
- durable organizational structures

These map to the concept of “ongoing commitments” that remain active without a defined endpoint <a name="forte-2022b-citation"></a>[(Forte, 2022)](#forte-2022b-reference).

## Guessing Resource Placement

Documents belong in `resources/` when they provide:

- reusable reference material
- explanations, definitions, or conceptual guides
- cross-project reusable knowledge
- tutorials or structured examples

Reference materials naturally fit here because they supply stable knowledge for multiple contexts <a name="marshall-2007-citation"></a>[(Marshall, 2007)](#marshall-2007-reference).

## Guessing Archive Placement

Documents belong in `archives/` when they:

- are inactive
- have historical value
- represent older versions of work
- have been superseded or replaced

Archival materials reflect the “cold storage” function common to information lifecycle models <a name="davenport-1997-citation"></a>[(Davenport, 1997)](#davenport-1997-reference).

## Strongest Heuristics

Across all PARA categories, the three strongest heuristics are:

- the document’s activity pattern
- the expected lifespan of the content
- whether the material is reusable across contexts

These principles mirror the foundations of information architecture and knowledge management systems <a name="morville-2006-citation"></a>[(Morville & Rosenfeld, 2006)](#morville-2006-reference).
